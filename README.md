# ISBI Multimodal Pathology Staging

This repository contains the training and pathology-report processing code used
for TCGA pathology staging experiments, with current focus on BRCA, KIRC, and LUSC.

The project has two main tracks:

- Multimodal WSI + text training for pathologic-stage binary classification.
- Pathology report preprocessing, ontology concept extraction, and text graph
  construction.

For the current workspace, use:

```text
Python: F:\Anaconda\envs\pytorch\python.exe
Repo:   F:\Tasks\isbi_code
Ontologies: F:\Tasks\Ontologies\processed
Hierarchy graphs: F:\Tasks\Pathology_Report_Hierarchy_Graphs\<BRCA|KIRC|LUSC>\<basic_hierarchy|ontology_concept_hierarchy|stage_keyword_word_hierarchy|stage_keyword_word_ontology_hierarchy>
```

## Entry Points

Project conventions for naming, organization, and temporary-file cleanup are in
`docs\project_conventions.md`. Codex/agent project instructions are mirrored in
`AGENTS.md`.

| Path | Purpose |
|---|---|
| `main.py` | Single-config train/test entry point. |
| `run_main_splits.py` | Multi-split training runner used by current experiments. |
| `train.py` | Main training implementation and model/eval integration. |
| `configs/config.yaml` | Hand-edited training config template. |
| `pathology_report_extraction/` | Report preprocessing, concept extraction, graph building, and manifests. |
| `tools/` | Experiment orchestration, analysis, data helpers, watchers, and runbooks. |
| `experiment_records/` | Stable, manually curated result summaries. |

## Supported Text Modes

| Mode | Meaning |
|---|---|
| `sentence_pt` | Original sentence feature sequence. |
| `hierarchy_graph` | Document/section/sentence hierarchy graph. |
| `concept_graph` | Hierarchy graph with ontology concept nodes. |
| `dual_text` | Raw sentence branch plus a graph/text branch with learned fusion. |

For graph-structured concept text, set `data.text_use_graph_structure: true`.
For the newest ablations, `dual_text` keeps raw sentence semantics and adds the
concept graph as a supplemental branch.
For concept-graph auxiliary runs, cap `model.text_dual_graph_weight_max` around
`0.2` and keep `loss.dual_text_graph_weight_target` around `0.1`. The current
ablation runner writes these auxiliary runs with an `auxgw20` experiment tag.

## Current Ontology Resources

The current ontology mainline is intentionally compact:

- NCIt: core cancer/task standardization.
- DO: disease hierarchy supplementation.

SNOMED CT and UMLS support remains in the legacy/explicit ablation path, but
new default experiments should use the `ncit_do` bundle unless a specific
cross-terminology recall experiment is being run.

The processed ontology root is:

```text
F:\Tasks\Ontologies\processed
```

The current NCIt+DO bundle is generated by:

```cmd
F:\Anaconda\envs\pytorch\python.exe pathology_report_extraction\build_ontology_ablation_bundles.py --variants ncit_do
```

## Common Commands

Run a single config:

```cmd
cd /d F:\Tasks\isbi_code
F:\Anaconda\envs\pytorch\python.exe main.py --config configs\config.yaml --mode train
```

Run the dual-text concept-graph ontology experiment with the current NCIt+DO
mainline:

```cmd
cd /d F:\Tasks\isbi_code
F:\Anaconda\envs\pytorch\python.exe tools\run_dual_text_concept_ablation.py --datasets KIRC BRCA --variants ncit_do --num_splits 3 --force_train
```

Watch that ablation:

```cmd
F:\Anaconda\envs\pytorch\python.exe F:\Tasks\isbi_code\tools\watch_dual_text_concept_ablation.py --once
```

## Latest Result Snapshot

The latest clean rerun summary is generated at:

```text
experiment_records\reports\ordered_splits\summary.md
```

Old result records were intentionally removed before the clean ordered rerun.
The ordered runner is append-style: rerunning the same dataset/method continues
from the next split in the same run folder, then renames the folder suffix to the
cumulative completed split count.
